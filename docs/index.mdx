<div className="max-w-4xl mx-auto px-4 py-8">
  <h1 className="text-4xl font-bold text-gray-900 mb-6">Welcome to EngramDB Documentation</h1>

  <p className="text-lg text-gray-700 mb-8">
    EngramDB is a specialized database system designed specifically for agent memory management. It provides efficient storage, retrieval, and querying of agent memories using a unified memory representation model.
  </p>

  <div className="bg-blue-50 p-6 rounded-lg mb-8">
    <h2 className="text-2xl font-semibold text-gray-900 mb-4">What is EngramDB?</h2>
    <p className="text-gray-700">
      EngramDB is a database system that combines vector, graph, and temporal properties in a single primitive called a MemoryNode. This unified approach allows for rich, multi-dimensional representation of agent memories, enabling complex queries that can leverage semantic similarity, relational connections, and temporal evolution.
    </p>
  </div>

  <h2 className="text-2xl font-semibold text-gray-900 mb-4">Core Features</h2>
  <ul className="grid grid-cols-1 md:grid-cols-2 gap-4 mb-8">
    <li className="bg-white p-4 rounded-lg shadow-sm">
      <span className="font-medium">Unified Memory Representation</span>: Combines graph, vector, and temporal properties in a single primitive
    </li>
    <li className="bg-white p-4 rounded-lg shadow-sm">
      <span className="font-medium">Vector Similarity Search</span>: Find memories with similar semantic content
    </li>
    <li className="bg-white p-4 rounded-lg shadow-sm">
      <span className="font-medium">Flexible Storage Options</span>: In-memory database for testing and development, file-based for persistence
    </li>
    <li className="bg-white p-4 rounded-lg shadow-sm">
      <span className="font-medium">Query API</span>: Rich querying with vector similarity, attribute filters, and temporal constraints
    </li>
    <li className="bg-white p-4 rounded-lg shadow-sm">
      <span className="font-medium">Memory Evolution</span>: Track changes to memories over time with temporal layers
    </li>
    <li className="bg-white p-4 rounded-lg shadow-sm">
      <span className="font-medium">Python Bindings</span>: First-class Python API for integration with ML and AI applications
    </li>
    <li className="bg-white p-4 rounded-lg shadow-sm">
      <span className="font-medium">Web Interface</span>: Browser-based UI for visualization and interaction with the database
    </li>
  </ul>

    </p>
    <ul className="space-y-2 text-gray-700">
      <li className="flex items-start">
        <span className="text-blue-500 mr-2">•</span>

      </li>
      <li className="flex items-start">
        <span className="text-blue-500 mr-2">•</span>
        <span><span className="font-medium">Graph databases</span> handle relationships well but aren't optimized for vector similarity</span>
      </li>
      <li className="flex items-start">
        <span className="text-blue-500 mr-2">•</span>
        <span><span className="font-medium">Document databases</span> provide flexible schemas but lack specialized memory features</span>
